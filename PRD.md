# HR AI 交流平台 — 产品设计文档 v1.0

> **项目代号**：待定
> **创建日期**：2026-05-18
> **目标**：搭建面向 HR 体系的 AI 知识共享与交流平台，通过飞书工作台 H5 网页应用接入

---

## 一、产品定位与背景

### 1.1 背景
集团 HR 团队正在推进 AI 在人力资源各模块的落地应用。当前痛点：
- AI 应用经验分散在个人手中，缺乏系统化沉淀
- HR 同事遇到 AI 相关问题不知道找谁、去哪找答案
- AI 活动（大赛、公开课等）缺乏统一承载平台，信息散落在群聊/文档中
- 没有一个「HR 的 AI 家园」来形成社群效应

### 1.2 一句话定位
> **HR 圈的 AI 社区** — 案例沉淀、知识学习、活动运营、问题互助的一站式平台

### 1.3 目标用户（分阶段）

| 阶段 | 用户范围 | 预估人数 |
|------|---------|---------|
| Phase 1 | HR、HRBP、COE | 100-300人 |
| Phase 2 | 扩展至集团全员 | 1000+人 |

### 1.4 核心价值主张
- **学**：AI 公开课、最佳实践全收录，随时学习
- **用**：AI 案例库 + 应用推荐，拿来就用
- **玩**：AI 大赛、挑战赛，以赛促学
- **问**：AI 问答社区 + 智能助手，有问必答

## 三、功能模块详细设计

### 3.1 首页 Dashboard

**目标**：打开即看到平台全貌，快速进入感兴趣的内容

**页面结构**：
- **顶部 Banner**：运营位轮播（重要活动、精选案例、公开课推荐）
- **数据看板**：平台活跃度概览（本周新增案例数、问答数、活跃用户数）
- **热门案例**：精选 3-5 篇置顶/热门案例卡片
- **进行中的活动**：当前 AI 大赛/活动卡片
- **最新问答**：最近 5 条热门问题
- **推荐课程**：最新/最热公开课推荐
- **快捷入口**：「我要投稿」「发起提问」「查看全部活动」

**参考标杆**：
- **掘金/InfoQ 首页** — 内容社区的经典 Dashboard 布局，Banner + 热门内容 + 分类入口
- **飞书知识库首页** — 左侧导航 + 右侧内容区的布局思路

---

### 3.2 AI 案例库（Case Library）

**目标**：系统化沉淀 HR 各模块的 AI 应用案例，让好经验可复用

#### 3.2.1 案例列表页
- **筛选维度**：HR 模块（招聘/培训/绩效/薪酬/员工关系/组织发展）、AI 工具类型、难度等级、排序方式
- **搜索**：支持关键词全文搜索
- **卡片展示**：封面图 + 标题 + 摘要 + 作者 + 点赞/收藏数 + 发布时间
- **视图切换**：卡片视图 / 列表视图

#### 3.2.2 案例详情页
- **基本信息**：标题、作者、所属模块、使用的 AI 工具、发布时间
- **正文区域**：富文本，支持图文混排、代码块、表格
- **互动区**：点赞、收藏、评论（支持嵌套回复）
- **相关推荐**：基于标签推荐相似案例
- **分享**：一键分享到飞书群聊/个人

#### 3.2.3 案例投稿（UGC）
- **发布流程**：填写标题 → 选择分类 → 撰写正文 → 添加标签 → 提交审核 → 发布
- **编辑器**：富文本编辑器，支持 Markdown、图片上传、附件
- **审核机制**：UGC 内容需管理员审核后可见（PGC 直接发布）
- **草稿箱**：支持保存草稿，断点续写

**参考标杆**：
- **阿里内部「阿里味儿」社区** — 案例分享 + 点赞评论的社区模式
- **Medium** — 优质的内容阅读体验，卡片式列表 + 沉浸式阅读
- **语雀** — 富文本编辑能力强，适合知识沉淀类内容

### 3.3 AI 应用推荐（App Showcase）

**目标**：帮助 HR 快速发现和使用适合自己的 AI 工具

#### 3.3.1 应用列表页
- **分类标签**：对话类（ChatGPT/Claude）、写作类、设计类、数据分析类、自动化类、HR 专属工具
- **卡片信息**：应用 Logo + 名称 + 简介 + 适用场景标签 + 评分 + 使用人数
- **筛选**：按场景、评分、热度排序

#### 3.3.2 应用详情页
- **应用介绍**：功能说明、适用场景、优缺点
- **使用指南**：图文/视频教程
- **快速入口**：直达链接（跳转到工具官网/内部部署地址）
- **用户评价**：评分 + 使用心得评论
- **相似推荐**：同类型工具对比

**参考标杆**：
- **Product Hunt** — 应用发现的经典模式，卡片 + 投票 + 评论
- **少数派 App 推荐** — 深度的使用评测和推荐
- **Dify 的 Plugin 市场** — AI 工具的分类和展示

### 3.4 AI 活动（Events & Competitions）

**目标**：承载 AI 大赛、挑战赛、Workshop 等活动的全流程运营

#### 3.4.1 活动列表页
- **状态分区**：进行中（高亮）→ 即将开始 → 已结束（可查看结果）
- **活动卡片**：封面 + 标题 + 时间 + 报名人数 + 状态标签
- **活动类型**：AI 大赛、AI 挑战赛、AI Workshop、AI 黑客松

#### 3.4.2 活动详情页
- **活动信息**：标题、时间、规则说明、奖项设置
- **报名/参与**：一键报名（飞书账号自动关联）
- **作品提交**：支持上传文件、填写表单、粘贴链接
- **排行榜/作品展示**：参赛作品公开展示，支持投票
- **活动动态**：公告、更新、FAQ
- **结果公示**：获奖名单、优秀作品精选

#### 3.4.3 活动管理后台
- **活动创建**：配置活动信息、规则、时间线
- **报名管理**：查看报名名单、导出
- **作品管理**：审核作品、评分、评奖
- **数据统计**：参与率、作品数、互动数据

**参考标杆**：
- **飞书举办的各类 AI 创新赛** — 企业内部 AI 大赛的运营模式
- **Kaggle 竞赛** — 竞赛 + 排行榜 + 作品展示的经典模式
- **Hackathon.io** — 黑客松活动管理平台

### 3.5 AI 公开课（Courses）

**目标**：AI 知识体系化沉淀，支持 HR 自主学习

#### 3.5.1 课程列表页
- **分类**：AI 基础认知、Prompt Engineering、HR 场景实操、工具教程、前沿趋势
- **课程卡片**：封面 + 标题 + 讲师 + 时长 + 学习人数 + 评分
- **筛选**：按分类、难度（入门/进阶/高阶）、形式（视频/文档/直播回放）

#### 3.5.2 课程详情页
- **课程信息**：标题、讲师介绍、课程大纲、目标受众
- **学习内容**：
  - 视频课程（嵌入飞书妙记/飞书视频播放器）
  - 文档类课程（富文本/PDF 在线阅读）
  - 直播回放
- **课程目录**：章节列表，支持按章节学习
- **互动区**：课程笔记、讨论区、问答
- **学习进度**：已学/未学标记，进度百分比

#### 3.5.3 学习中心（个人维度）
- **我的课程**：已报名/学习中的课程列表
- **学习进度**：各课程完成百分比
- **学习路径**：推荐的学习顺序（可选功能）
- **证书/徽章**：完成课程后的成就标识（可选功能）

**参考标杆**：
- **极客时间** — 课程目录 + 学习进度 + 笔记讨论的完整体验
- **飞书妙记** — 视频内容的组织和播放
- **LinkedIn Learning** — 企业学习平台的课程分类和推荐逻辑
- **得到 App** — 学习进度追踪 + 成就体系

### 3.6 AI 问答社区（Q&A Community）

**目标**：HR 遇到 AI 相关问题时，有地方问、有人答、答案可沉淀

#### 3.6.1 问题列表页
- **排序**：最新、最热、未回答、已采纳
- **标签筛选**：按问题类型标签筛选
- **问题卡片**：标题 + 标签 + 回答数 + 浏览数 + 最后活跃时间

#### 3.6.2 问题详情页
- **问题描述**：标题 + 详细描述（富文本）+ 标签
- **回答列表**：
  - 支持多个回答
  - 投票（赞同/反对），高赞回答排序靠前
  - 提问者可采纳最佳回答
  - 支持评论追问
- **相关问题**：相似问题推荐，避免重复提问

#### 3.6.3 AI 智能问答（接入 LLM）
- **入口**：问答区顶部的「AI 助手」Tab，或独立入口
- **交互方式**：对话式，用户输入问题 → AI 回答
- **能力边界**：
  - 基于平台已有案例/课程/问答内容做 RAG 增强
  - 回答时标注引用来源
  - 支持追问和多轮对话
- **人机协作**：AI 回答后，用户可标记「有帮助/无帮助」，无帮助的问题可转人工社区提问

**参考标杆**：
- **Stack Overflow** — Q&A 的经典模式，投票 + 采纳 + 标签
- **知乎** — 问答 + 社区氛围，适合企业内部场景
- **Dify/Coze 的 AI 对话** — LLM 对话交互的 UI 参考
- **GitBook AI** — 基于知识库的 RAG 问答

## 四、技术架构

### 4.1 整体架构

```
┌─────────────────────────────────────────────┐
│                  飞书工作台                    │
│            (H5 网页应用入口)                   │
└──────────────────┬──────────────────────────┘
                   │ Web View
┌──────────────────▼──────────────────────────┐
│              前端 (H5 Web App)                │
│         Vue3/React + 移动端适配                │
└──────────────────┬──────────────────────────┘
                   │ API
┌──────────────────▼──────────────────────────┐
│              后端服务层                        │
│    Node.js/Python + RESTful API              │
├──────────────────────────────────────────────┤
│  内容服务 │ 用户服务 │ 活动服务 │ AI 服务      │
└──────────────────┬──────────────────────────┘
                   │
┌──────────────────▼──────────────────────────┐
│              数据层                            │
│  PostgreSQL │ Redis │ 对象存储(OSS/MinIO)     │
└──────────────────┬──────────────────────────┘
                   │
┌──────────────────▼──────────────────────────┐
│              外部集成                          │
│  飞书开放平台(LLM API)                        │
└──────────────────────────────────────────────┘
```

### 4.2 飞书集成方案

| 能力 | 集成方式 | 说明 |
|------|---------|------|
| 身份认证 | 飞书 OAuth2.0 | 免登，飞书账号直接登录 |
| 消息通知 | 飞书机器人消息 | 评论、点赞、活动提醒推送到飞书 |
| 内容分享 | 飞书分享卡片 | 案例/课程一键分享到飞书群聊 |
| 组织架构 | 飞书通讯录 API | 自动获取用户部门、角色信息 |
| 应用入口 | 飞书工作台 | 在飞书工作台添加应用图标，点击打开 H5 |

### 4.3 AI 能力架构

```
用户提问
   │
   ▼
意图识别（是否需要 AI 回答）
   │
   ├─ 常规社区提问 → 走普通 Q&A 流程
   │
   └─ AI 回答 → RAG Pipeline
                    │
        ┌───────────┼───────────┐
        ▼           ▼           ▼
   案例库检索    课程库检索    历史问答检索
        │           │           │
        └───────────┼───────────┘
                    ▼
              拼装 Prompt
              (系统提示 + 检索结果 + 用户问题)
                    │
                    ▼
              LLM 生成回答
              (Claude / GPT / 内部模型)
                    │
                    ▼
              返回回答 + 引用来源
```

### 4.4 推荐技术栈

| 层级 | 推荐方案 | 理由 |
|------|---------|------|
| 前端 | **Next.js (React)** 或 **Nuxt.js (Vue3)** | SSR/SSG 友好，SEO 不是重点但性能好；Vue 对 HR 团队可能更友好 |
| UI 组件库 | **Ant Design** / **Arco Design** | 企业级组件库，表单/表格/卡片等开箱即用 |
| 后端 | **Node.js (Nest.js)** 或 **Python (FastAPI)** | Nest.js 全栈统一语言；FastAPI 若 AI 逻辑重则更合适 |
| 数据库 | **PostgreSQL** | 结构化数据 + JSON 字段灵活性 |
| 缓存 | **Redis** | 热门内容缓存、排行榜、会话 |
| 对象存储 | **OSS / MinIO** | 图片、视频、附件存储 |
| 搜索 | **PostgreSQL Full-Text** 或 **MeiliSearch** | 一期用 PG 全文搜索，二期可升级 MeiliSearch |
| AI | **Claude API** / **内部模型 API** | RAG 检索 + LLM 生成 |

> **Vibe Coding 建议**：如果追求快速出 MVP，推荐 **Next.js + Supabase（PostgreSQL）+ Vercel 部署**，Supabase 自带认证、存储、实时订阅，极大减少后端开发量。

## 七、数据模型（核心表）

```sql
-- 用户表（飞书 OAuth 同步）
users (id, feishu_open_id, name, avatar, department, role, bio, created_at)

-- 案例表
cases (id, title, summary, content, cover_image, category, ai_tools[], difficulty,
       author_id, status(draft/pending/published/rejected), view_count, like_count,
       bookmark_count, created_at, updated_at)

-- 应用推荐表
apps (id, name, logo, description, category, scenarios[], official_url,
      rating, user_count, created_at)

-- 课程表
courses (id, title, description, cover_image, instructor, category, difficulty,
         duration, content_type(video/doc/live), syllabus_json, student_count,
         rating, created_at)

-- 课程章节表
course_chapters (id, course_id, title, content_url, duration, sort_order)

-- 学习记录表
learning_records (id, user_id, course_id, chapter_id, progress, completed_at)

-- 活动表
events (id, title, description, cover_image, type(competition/workshop/hackathon),
        rules, prizes, start_time, end_time, registration_deadline,
        status(upcoming/ongoing/ended), max_participants, created_at)

-- 活动报名表
event_registrations (id, event_id, user_id, registered_at)

-- 活动作品表
event_submissions (id, event_id, user_id, title, content, file_url,
                   score, rank, submitted_at)

-- 问题表
questions (id, title, content, tags[], author_id, view_count, answer_count,
           has_accepted_answer, created_at)

-- 回答表
answers (id, question_id, content, author_id, vote_count, is_accepted,
         created_at)

-- 评论表（案例/课程/回答通用）
comments (id, target_type, target_id, content, author_id, parent_id, created_at)

-- 点赞表
likes (id, user_id, target_type, target_id, created_at)

-- 收藏表
bookmarks (id, user_id, target_type, target_id, created_at)

-- 通知表
notifications (id, user_id, type, title, content, target_type, target_id, is_read, created_at)
```

## 九、运营策略建议

### 9.1 冷启动策略
1. **种子内容**：平台上线前，由 HR COE 团队预写 20-30 篇高质量案例
2. **邀请制**：先邀请 20-30 位 HR 核心用户内测，收集反馈
3. **首发活动**：以一场 AI 大赛作为平台上线的引爆点
4. **公开课引流**：将已有 AI 公开课内容首批导入

### 9.2 持续运营
- **每周精选**：每周评选 Top 案例/回答，推送到飞书群
- **月度排行榜**：最活跃贡献者、最佳案例、最佳回答
- **AI 挑战赛**：定期举办小规模 AI 实践挑战
- **内容运营日历**：规划每周/每月的内容发布节奏

### 9.3 激励机制
- **积分体系**：发布案例/回答问题/被采纳 获得积分
- **等级/徽章**：AI 新手 → AI 探索者 → AI 达人 → AI 专家
- **荣誉展示**：个人主页展示贡献度和成就
