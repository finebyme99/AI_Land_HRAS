## 一、产品定位与背景

### 1.1 背景
集团 HR 团队正在推进 AI 在人力资源各模块的落地应用。当前痛点：
- AI 应用经验分散在个人手中，缺乏系统化沉淀
- HR 同事遇到 AI 相关问题不知道找谁、去哪找答案
- AI 活动（大赛、公开课等）缺乏统一承载平台，信息散落在群聊/文档中
- 没有一个「HR 的 AI 家园」来形成社群效应

### 1.2 一句话定位
> **HR 圈的 AI 社区** — 案例沉淀、知识学习、活动运营、话题互助的一站式平台

### 1.3 目标用户
面向集团 HR、HRBP、COE，预估 100-300 人。

### 1.4 核心价值主张
- **学**：AI 公开课、最佳实践全收录，随时学习
- **用**：AI 案例库 + 应用推荐，拿来就用
- **玩**：AI 大赛，以赛促学
- **聊**：话题社区，有问必答

## 三、功能模块详细设计

### 3.1 首页 Dashboard

**目标**：打开即看到平台全貌，快速进入感兴趣的内容

**页面结构**：
- **顶部 Banner**：运营位轮播（重要活动、精选案例、公开课推荐）
- **数据看板**：平台活跃度概览（本周新增案例数、话题数、活跃用户数）
- **热门案例**：精选 3-5 篇置顶/热门案例卡片
- **进行中的活动**：当前 AI 大赛卡片
- **最新话题**：最近 5 条热门话题
- **推荐课程**：最新/最热公开课推荐
- **快捷入口**：「我要投稿」（提示：平台投稿支持关联AI大赛，可选择是否同步参加AI大赛，默认自动参赛）「发起话题或提问」「查看全部活动」

**参考标杆**：
- **掘金/InfoQ 首页** — 内容社区的经典 Dashboard 布局，Banner + 热门内容 + 分类入口
- **飞书知识库首页** — 左侧导航 + 右侧内容区的布局思路

---

### 3.2 AI 案例库（Case Library）

**目标**：系统化沉淀 HR 各模块的 AI 应用案例，让好经验可复用

#### 3.2.1 案例列表页
- **筛选维度**：HR 模块（招聘/培训/绩效/薪酬/员工关系/组织发展）、AI 工具类型、难度等级、排序方式
- **搜索**：支持关键词全文搜索
- **卡片展示**：封面图 + 标题 + 摘要 + 作者 + 点赞/收藏数 + 发布时间
- **视图切换**：卡片视图 / 列表视图

#### 3.2.2 案例详情页
- **基本信息**：标题、作者、所属模块、使用的 AI 工具、发布时间
- **正文区域**：富文本，支持图文混排、代码块、表格
- **互动区**：点赞、收藏、评论（支持嵌套回复）
- **相关推荐**：基于标签推荐相似案例
- **分享**：一键分享到飞书群聊/个人

#### 3.2.3 案例投稿（UGC）
- **发布流程**：填写标题 → 选择分类 → 撰写正文 → 添加标签 → 选择是否关联 AI 大赛（默认自动参赛）→ 提交审核 → 发布
- **编辑器**：富文本编辑器，支持 Markdown、图片上传、附件
- **审核机制**：UGC 内容需管理员审核后可见（PGC 直接发布）
- **草稿箱**：支持保存草稿，断点续写

**参考标杆**：
- **阿里内部「阿里味儿」社区** — 案例分享 + 点赞评论的社区模式
- **Medium** — 优质的内容阅读体验，卡片式列表 + 沉浸式阅读
- **语雀** — 富文本编辑能力强，适合知识沉淀类内容

### 3.3 AI 大赛（Competition）

**目标**：承载 AI 大赛的全流程运营（目前只有 AI 大赛，第一赛季进行中）

#### 3.3.1 大赛列表页
- **状态分区**：进行中（高亮）→ 即将开始 → 已结束（可查看结果）
- **活动卡片**：封面 + 标题 + 时间 + 报名人数 + 状态标签

#### 3.3.2 大赛详情页
- **活动信息**：标题、时间、规则说明、奖项设置
- **报名/参与**：关联提报链接
- **作品提交**：支持上传文件、填写表单、粘贴链接
- **排行榜/作品展示**：参赛作品公开展示，支持投票
- **活动动态**：公告、更新、FAQ
- **结果公示**：获奖名单、优秀作品精选

**参考标杆**：
- **飞书举办的各类 AI 创新赛** — 企业内部 AI 大赛的运营模式
- **Kaggle 竞赛** — 竞赛 + 排行榜 + 作品展示的经典模式

### 3.4 AI 公开课（Courses）

**目标**：AI 知识体系化沉淀，支持 HR 自主学习

#### 3.4.1 课程列表页
- **分类**：AI工具基础、HR场景实操、AI工具进阶
- **课程卡片**：封面 + 标题 + 讲师 + 时长 + 学习人数 + 评分
- **筛选**：按分类、难度（入门/基础/进阶）、形式（视频/文档）

#### 3.4.2 课程详情页
- **课程信息**：标题、讲师介绍、课程大纲、目标受众
- **学习内容**：
  - 视频课程（嵌入飞书视频链接，支持管理员上传配置）
  - 文档类课程（飞书云文档）
- **课程目录**：章节列表，支持按章节学习
- **互动区**：课程笔记、讨论区、问答
- **学习进度**：已学/未学标记，进度百分比

**参考标杆**：
- **极客时间** — 课程目录 + 学习进度 + 笔记讨论的完整体验
- **LinkedIn Learning** — 企业学习平台的课程分类和推荐逻辑

### 3.5 AI 话题社区（Topics & Q&A）

**目标**：HR 遇到 AI 相关问题时，有地方聊、有人答、答案可沉淀

#### 3.5.1 话题列表页
- **排序**：最新、最热、未回答、已采纳
- **标签筛选**：按话题类型标签筛选
- **话题卡片**：标题 + 标签 + 回答数 + 浏览数 + 最后活跃时间

#### 3.5.2 话题详情页
- **话题描述**：标题 + 详细描述（富文本）+ 标签
- **回答列表**：
  - 支持多个回答
  - 投票（赞同/反对），高赞回答排序靠前
  - 提问者可采纳最佳回答
  - 支持评论追问
- **相关话题**：相似话题推荐，避免重复提问

**参考标杆**：
- **Stack Overflow** — Q&A 的经典模式，投票 + 采纳 + 标签
- **知乎** — 问答 + 社区氛围，适合企业内部场景

### 3.6 AI 应用推荐（App Showcase）

**目标**：帮助 HR 快速发现和使用适合自己的 AI 工具

#### 3.6.1 应用列表页
- **分类标签**：对话类（ChatGPT/Claude）、写作类、设计类、数据分析类、自动化类、HR 专属工具
- **卡片信息**：应用 Logo + 名称 + 简介 + 适用场景标签 + 评分 + 使用人数
- **筛选**：按场景、评分、热度排序
- **互动**：支持点赞、点踩、评论

#### 3.6.2 应用详情页
- **应用介绍**：功能说明、适用场景、优缺点
- **使用指南**：图文/视频教程
- **快速入口**：直达链接（跳转到工具官网/内部部署地址）
- **用户评价**：评分 + 使用心得评论
- **相似推荐**：同类型工具对比

#### 3.6.3 应用投稿（UGC）
- 支持用户分享自己认为好用的应用
- 管理员审核通过即可发布

**参考标杆**：
- **Product Hunt** — 应用发现的经典模式，卡片 + 投票 + 评论
- **少数派 App 推荐** — 深度的使用评测和推荐

```
┌─────────────────────────────────────────────┐
│                  飞书工作台                    │
│            (H5 网页应用入口)                   │

                   │ Web View
┌──────────────────▼──────────────────────────┐
│              前端 (H5 Web App)                │

└──────────────────┬──────────────────────────┘
                   │ API
┌──────────────────▼──────────────────────────┐
│              Supabase BaaS                   │
│  认证 │ 数据库(PostgreSQL) │ 存储 │ 实时订阅  │
├──────────────────────────────────────────────┤
│  内容服务 │ 用户服务 │ 活动服务                │
└──────────────────┬──────────────────────────┘
                   │
┌──────────────────▼──────────────────────────┐
│              外部集成                          │
│  飞书开放平台 │ 国产大模型(OpenAI协议)         │
└──────────────────────────────────────────────┘
```

### 4.2 飞书集成方案

| 能力 | 集成方式 | 说明 |
|------|---------|------|
| 身份认证 | 飞书 OAuth2.0 | 免登，飞书账号直接登录，自动获取头像和用户名 |
| 消息通知 | 飞书机器人消息 | 评论、点赞、活动提醒推送到飞书 |
| 内容分享 | 飞书分享卡片 | 案例/课程一键分享到飞书群聊 |
| 组织架构 | 飞书通讯录 API | 自动获取用户部门、角色信息 |
| 应用入口 | 飞书工作台 | 在飞书工作台添加应用图标，点击打开 H5 |

### 4.3 技术栈（已确认）

| 层级 | 方案 | 说明 |
|------|------|------|
| 前端 | **Next.js 16 (React 19)** | 用户易上手、开发轻量快速 |
| UI 组件库 | **Ant Design 6** | 企业级组件库 |
| 后端 | **Supabase BaaS** | 自带认证、PostgreSQL、存储、实时订阅 |
| 数据库 | **Supabase PostgreSQL** | 结构化数据 + JSON 字段灵活性 |
| 对象存储 | **Supabase Storage** | 图片、视频、附件存储 |
| 搜索 | **PostgreSQL Full-Text** | 一期用 PG 全文搜索 |
| AI | **国产大模型（OpenAI 协议接入）** | 自定义国内模型 |
| 多语言 | **next-intl 或类似方案** | 支持中英双语 |

## 八、关键交互流程

### 8.1 UGC 投稿审核流程

```
用户撰写案例/应用推荐 → 提交审核 → 管理员审核队列
                                      │
                              ┌───────┼───────┐
                              ▼       │       ▼
                           通过       │     拒绝
                     (自动发布+通知)   │  (通知+原因)
                                      │
                              ┌───────┘
                              ▼
                          修改建议
                    (通知用户修改后重新提交)
```

### 8.2 投稿关联大赛流程

```
用户撰写案例 → 选择是否关联 AI 大赛
                    │
            ┌───────┼───────┐
            ▼               ▼
     自动参赛(默认)      仅发布案例
            │
            ▼
    案例同时提交到大赛作品池
```

### 8.3 AI 大赛运营流程

```
管理员创建大赛 → 配置规则/奖项/时间线 → 发布上线
                                              │
                    ┌─────────────────────────┤
                    ▼                         ▼
              用户浏览/报名              飞书消息推送
                    │                    (活动提醒)
                    ▼
              提交作品(截止前)
                    │
                    ▼
              评审/投票/打分
                    │
                    ▼
              结果公示 + 颁奖
                    │
                    ▼
              优秀作品收录到案例库
```

## 九、运营策略建议

### 9.1 冷启动策略
1. **种子内容**：平台上线前，由 HR COE 团队预写 20-30 篇高质量案例
2. **邀请制**：先邀请 20-30 位 HR 核心用户内测，收集反馈
3. **首发活动**：以第一赛季 AI 大赛作为平台上线的引爆点
4. **公开课引流**：将已有 AI 公开课内容首批导入

### 9.2 持续运营
- **每周精选**：每周评选 Top 案例/回答，推送到飞书群
- **月度排行榜**：最活跃贡献者、最佳案例、最佳回答
- **内容运营日历**：规划每周/每月的内容发布节奏

### 9.3 激励机制
- **积分体系**：发布案例/回答问题/被采纳 获得积分
- **等级/徽章**：AI 新手 → AI 探索者 → AI 达人 → AI 专家
